JavaScript-Program Structure

30Nov2021

Program Structure as the name itself suggests that program in structure that is we have know about programs creating values and applying operators to create new values but they need to be put in larger structure to be useful.

Expressions and Statements

Expressions- fragment of code that produces value.Expression can be literally anything they are kind of very similar to the sentences like a sentence can have sub-sentences and a sub-sentence have another sub-sentence and so on this is how a expression can have sub-expressions and so on this way it allows to write arbitrarily complex computations.For ex-"Akshat","Expressions","#id".

Statements- We can simply say a Statement as a Expression having semicolon after it.For ex-button.addEventListener("click",function);.Expression corresponds to a sentence fragment and statement corresponds to full statement.An Expression can be content to just produce value.A statement amounts to something only if it affects the code like the example it helps to select the buttons click event .Such changes which affects the program like in example are called Side-effects

Bindings / Variables

Lets simplify it they are sort of variables used in Maths and to understand them properly they are like place-holders to catch or to define values specifically or you can think of a specific drawer in which specific things are kept inside it.There are 3 ways to create a Binding that is Let,Const,Var(Let,Constant,Variable,) Every Binding starts with them followed up by a name and = operator then a value .For ex-let x=0 , const name=Akshat. Imagine like a pen holder that can keep only one pen every time so in var/let case you can access the pen till its not changed with another pen otherwise you can access the pen but the pen which you will access is that another pen not the same so the values which once given to a binding can only be access till its not changed otherwise the changed value will be accessed in case of const you can't keep a different pen once const binding given a value it will remain unchanged in the whole program. Binding Nomenclature it can be any word with or without numeric characters without any special character and reserved-keywords except $ and _

The Environment

Bindings and values collectively that exists at point of time is called Environment . By default its not empty.It always have language standard Bindings.

Functions

Function can be explained as small machines do their programs and gives their values as outputs and/or creates side-effects which can be further used. They basically are piece of program wrapped in a value.Executing one is called the invoking,calling,applying.They are wrote like name and parentheses after that like functionName(arguments). Arguments are the values that are passed while executing function and Parameters are given while describing a function . Like-function(parameters){}.

Console.log Function

Basically this function helps us to access the resultant value passed as argument to it and can be analysed in our browsers console.Its not regular function since it access the log binding from the console binding.

Control-Flow

Programs are always executed like they are story from top to bottom line wise. Number(),Boolean(),String() are the functions which helps to convert the value in the form as their respective name suggests when passed as arguments.

Conditional-Execution

All programs don't have straight control flow there can be branching road like conditions like either this or no this or anything else or nothing at all if condition is not fulfilled.This is called Conditional Execution This is achieved through if and else statements and also If/Else after which a block is defined through {} these braces in which multiple line of codes can be written known as a Block

While and Do loops

Idea of writing program is to make things work with less effort.Sometimes we need to do a repetitive task having some pattern so we use a control flow known as Loop. While loop is a loop which will repeat the whole Block of code until the condition is fulfilled which has been passed into it .A do loop is a control structure which will execute itself at least once

Comments

We sometimes want to make the understand somethings to code reader which cant't always be studied by reading code so we use comments in this case .The text or code commented doesn't shows to client Comments written in two ways

  • // This goes only to the end of line
  • /*---*/ Any section of text irrespective of the size of text inside these will be commented.

Summary

We can store values with the help of bindings and not only that we can also write deliberate functions with the help of loops ,bindings and also If statements.These things are a lot useful in day to day coding.
